# 堆、栈、队列之间的区别

  • 堆：堆是在程序运行时，而不是程序编译时，申请某个大小的内存空间（动态分配内存），对其访问和对一般内存访问没有区别
  • 栈：一个桶，后放进去的先出来，下面的东西要等前面的出来之后才能出现
  • 队列：只能在队头做删除操作，在队尾做插入操作，而栈只能在栈顶做插入与删除（先进先出）

# 正向代理和反向代理的区别

  • 正向代理：位于客户端和服务端的代理服务器，客户端向代理服务器发送请求，并指定目标服务端，然后代理服务器会去代理请求，向目标服务器请求获得响应后并返回给客户端
  • 反向代理：与正向恰好相反，对于客户端来说，代理服务器就像目标服务器，客户端直接访问代理服务器，代理服务器会转发到对应目标服务器，客户端并不知道谁是真正的目标服务器

区别

正向代理代理的是客户端，帮客户端发送请求给服务器，服务器响应后，由代理服务器返回内容给客户端，比较常见的场景就是翻墙

反向代理在生产环境用的比较多，代理服务器代理目标服务器，负责转发请求，目标服务器对客户端是不可见的，不过客户端请求目标服务器和请求代理是一样的效果，常见的场景的 Nginx 的负载均衡

# CPython 一些宏定义

  • _Py_NewReference：接收一个对象，将其引用计数设置为 1，用于新创建的对象。此外我们在定义里面还看到了一个宏 Py_REFCNT，这是用来获取对象引用计数的，也就是获取 ob_refcnt 成员的值。还有 Py_TYPE，这是专门获取对象的类型值，得到的是一个指向 PyTypeObject 结构体实例的指针，也就是 ob_type 成员的值
  • _Py_Dealloc：接收一个对象，执行该对象的类型对象里的析构函数，对该对象进行回收
  • Py_INCREF：接收一个对象，将该对象的引用计数 + 1，即 ob_refcnt++
  • Py_DECREF：接收一个对象，将该对象的引用计数 - 1，即 ob_refcnt–，如果 - 1 后发现为 0，就会调用_Py_Dealloc
  • Py_XINCREF：和 Py_INCREF 一致，但可以处理空指针
  • Py_XDECREF：和 Py_DECREF 一致，但可以处理空指针
  • Py_CLEAR：和 Py_XDECREF 类似，也可以处理空指针

当一个对象引用计数为 0 时，该对象的析构函数就会调用，然后回收对象、销毁对象、删除对象等，但不代表最终一定会 free 掉，对象占用内存可能还在

如果占用的内存也释放掉，那么就是频繁的申请、释放内存，大大影响 Python 的执行效率，所以在 CPython 中，大量使用了缓存池，只要缓存池没有满，就不会释放内存，直到缓存池满了，才会真正的释放内存

# CPython PyTypeObject

在 PyTypeObject 中，有三个非常重要的字段

名称 对应函数族 作用
tp_as_number PyNumberMethods 定义了 Python 对象的行为可以像数字类型执行加减乘除等操作
tp_as_sequence PySequenceMethods 定义了 Python 对象作为一个顺序表一样的行为，如 list
tp_as_mapping PyMappingMethods 定义了 Python 对象的关联行为，如 dict
